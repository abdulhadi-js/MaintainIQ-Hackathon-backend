# MaintainIQ Backend
[![Deploy Backend to AWS](https://github.com/abdulhadi-js/MaintainIQ-Hackathon-backend/actions/workflows/deploy.yml/badge.svg)](https://github.com/abdulhadi-js/MaintainIQ-Hackathon-backend/actions/workflows/deploy.yml)

The backend service for MaintainIQ, built with **NestJS**, **TypeORM**, and **PostgreSQL**. It powers the AI-driven triage system, asset management, and role-based workflows for technicians and supervisors.

## Tech Stack
* **Framework:** NestJS
* **Database:** PostgreSQL with TypeORM
* **Authentication:** JWT (JSON Web Tokens) with Role-Based Access Control (RBAC)
* **AI Integration:** Google Generative AI (Gemini) for issue triage

## Prerequisites
* Node.js (v18 or higher recommended)
* PostgreSQL database

## Installation

1. Install dependencies:
   ```bash
   npm install
   ```

2. Environment Variables:
   Create a `.env` file in the root directory:
   ```env
   PORT=3000
   NODE_ENV=development

   # Database configuration
   DB_HOST=localhost
   DB_PORT=5432
   DB_USERNAME=postgres
   DB_PASSWORD=postgres
   DB_DATABASE=maintainiq
   DB_SSL=false

   # JWT
   JWT_SECRET=your_jwt_secret_key_change_in_production
   JWT_EXPIRES_IN=7d

   # URLs
   FRONTEND_URL=http://localhost:5173

   # Google Gemini API
   GEMINI_API_KEY=your_gemini_api_key_here
   ```

## Running the App

```bash
# development
npm run start

# watch mode
npm run start:dev

# production mode
npm run start:prod
```

## API Documentation & Endpoints
Once the server is running, you can access the full OpenAPI (Swagger) documentation automatically generated by NestJS at:

**Swagger UI URL:** `http://localhost:3000/api/docs`

### Major REST Endpoints (Prefixed by `/api/v1`)
* **Auth**
  * `POST /auth/login` - Authenticate and retrieve JWT
  * `POST /auth/register` - Create team members (Admin only)
* **Assets**
  * `GET /assets` - Fetch all assets
  * `POST /assets` - Create new asset
  * `GET /assets/public/:code` - Fetch minimal asset data for public QR viewing
* **Issues**
  * `POST /issues` - Public endpoint to report a new issue
  * `GET /issues` - List issues (filtered by status/assignee based on role)
  * `PATCH /issues/:id/status` - Update issue status (e.g. to `Closed` or `Reopened`)
  * `POST /issues/:id/assign` - Assign an issue to a technician
* **Technicians & Users**
  * `GET /users/technicians` - Get list of available technicians for assignment
* **Maintenance**
  * `POST /maintenance` - Record maintenance work, mark an issue as `Resolved`, and close out the technician's workflow
* **AI / Triage**
  * `POST /ai/triage` - Analyzes user descriptions and suggests categories/priorities
* **Uploads**
  * `POST /uploads` - Handle Base64 image transfers from public issue reports

## Seed Data
To test the supervisor flow or view pre-populated dashboards, you can use the built-in database seeder script:
```bash
node seed.js
```
This inserts synthetic assets, users, and successfully resolved issues directly into the PostgreSQL database.
