# MaintainIQ Backend

The backend service for MaintainIQ, built with **NestJS**, **TypeORM**, and **PostgreSQL**. It powers the AI-driven triage system, asset management, and role-based workflows for technicians and supervisors.

## Tech Stack
* **Framework:** NestJS
* **Database:** PostgreSQL with TypeORM
* **Authentication:** JWT (JSON Web Tokens) with Role-Based Access Control (RBAC)
* **AI Integration:** Google Generative AI (Gemini) for issue triage

## Prerequisites
* Node.js (v18 or higher recommended)
* PostgreSQL database

## Installation

1. Install dependencies:
   ```bash
   npm install
   ```

2. Environment Variables:
   Create a `.env` file in the root directory:
   ```env
   PORT=3000
   NODE_ENV=development

   # Database configuration
   DB_HOST=localhost
   DB_PORT=5432
   DB_USERNAME=postgres
   DB_PASSWORD=postgres
   DB_DATABASE=maintainiq
   DB_SSL=false

   # JWT
   JWT_SECRET=your_jwt_secret_key_change_in_production
   JWT_EXPIRES_IN=7d

   # URLs
   FRONTEND_URL=http://localhost:5173

   # Google Gemini API
   GEMINI_API_KEY=your_gemini_api_key_here
   ```

## Running the App

```bash
# development
npm run start

# watch mode
npm run start:dev

# production mode
npm run start:prod
```

## API Documentation
Once the server is running, you can access the full OpenAPI (Swagger) documentation automatically generated by NestJS.

**Swagger UI URL:** `http://localhost:3000/api/docs`

The API features endpoints divided by:
* `Auth` - Login, registration
* `Assets` - QR generation, lookup, creation
* `Issues` - Public reporting, assignment
* `Technicians` - Workflows and task management
* `Maintenance` - Recording logs and parts used
* `Supervisor` - Quality review and ticket resolution

## Seed Data
To test the supervisor flow or view pre-populated dashboards, you can use the built-in database seeder script:
```bash
node seed.js
```
This inserts synthetic assets, users, and successfully resolved issues directly into the PostgreSQL database.
